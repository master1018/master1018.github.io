Python搭建文献自动获取脚本

2023/7/1 爬虫 streamlit selenium pdfminer

脚本基于streamlit搭建前端进行交互，使用pdfminer读取paper，获取其中的文献信息，再用selenium去获取其bibTex和pdf文件链接，最后使用request方法进行下载。

仓库地址：https://github.com/master1018/auto_references_get

脚本尚存在一些问题，如速率的问题（因为是对google scholar进行爬取，理论上不开VPN进不去），以及一些文件可能下载不成功，有待改进。

下面是对整个脚本的一个介绍。

PDF读取

关于这一部分，主要使用pdfminer进行获取，最终目的就是获得pdf中References部分的所有内容，

def get_references(paper_name, target_str="references"):
    fp = open(paper_name, 'rb')
    parser = PDFParser(fp)
    rsrcmgr = PDFResourceManager()
    laparams = LAParams()
    device = PDFPageAggregator(rsrcmgr, laparams=laparams)
    interpreter = PDFPageInterpreter(rsrcmgr, device)
    document = PDFDocument(parser)

    text_content = []
    for page in PDFPage.create_pages(document):
        interpreter.process_page(page)
        layout = device.get_result()
        for lt_obj in layout:
            if isinstance(lt_obj, LTTextBox) or isinstance(lt_obj, LTTextLine):
                text_content.append(lt_obj.get_text())
            else:
                pass


    # text_content 中每一个元素存储了一行文字
    total_text = ''.join(text_content).replace("\n","")
    start = len(total_text) - 1
    while start > 0:
        if total_text[start: start + 10].lower() == target_str.lower():
            print(total_text[start: start + 10])
            break
        else:
            start -= 1
    file = open("./output/tmp_output","w")
    write_data = total_text[start: len(total_text)]
    len_data = len(write_data)
    cur = 0
    while cur < len_data:
        tail = cur + 1
        while tail < len_data and write_data[tail] != '[':
            tail += 1
        write_line = write_data[cur: tail]
        file.write(write_line + "\n")
        cur = tail

    file.close()

检测手法比较粗暴，直接当成大字符串处理，遇到Reference就认为后面的所有内容都是参考文献，并根据
“[“和”]”符号进行分段。

Google Scholar爬取

该部分应该就是核心地方了，使用了selenium进行网页的爬取。在这里，使用的网站是谷歌学术，如果有需要可以自行更换，不过发现在使用国内镜像熊猫学术的时候，爬取过程会出现超时导致的一系列问题，于是还是换了谷歌学术（可能和我开着梯子有关）。

def paperUrl(name):

    params = {
        'q':q

    params = parse.urlencode(params)
    url = "https://scholar.google.com/scholar?" + params

pdf文件链接获取

经过调研，在网页中，pdf文件都有一个href，但如果识别出一个链接为pdf文件的链接呢？最简单的思路是匹配链接字符串中有无pdf子串。但后面发现，有的链接没有pdf，因此该方法不妥。用开发者工具仔细查看，可以发现，如果一个链接是pdf文件，这个html节点的元素一定有一个“[PDF]“的类名，如下所示：

<a href="https://www.cs.utexas.edu/users/sniekum/classes/RLFD-F15/papers/Knox09.pdf" data-clk="hl=zh-CN&amp;sa=T&amp;oi=gga&amp;ct=gga&amp;cd=0&amp;d=13257701720605127865&amp;ei=eB-gZPCtGoqA6rQPw7eQmAo" data-clk-atid="ufTA-njR_LcJ">
  <span class="gs_ctg2">[PDF]</span> 
  utexas.edu
</a>

很显然，只要能匹配到[PDF]这个类名就行了，因此匹配过程如下，直接匹配每一个带有href的节点。

pdf文件下载

下载主要使用了requests方法，比较简单。

def down_load_paper(title, url):
    response = requests.get(url, stream=True)
    #title = "./output/paper/" + title
    # 检查响应状态码是否为200，表示请求成功
    time.sleep(5)
    if response.status_code == 200:
        # 以二进制写入模式打开文件
        with open("./output/paper/" + title, 'wb') as f:
            for chunk in response.iter_content(chunk_size=1024):
                f.write(chunk)
            print('PDF 文件已下载到本地')
    else:
        print('请求失败')
